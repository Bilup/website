Bilup 已于**2026年4月**开放了拓展库的提交通道。
为了确保拓展的质量及相关性，Bilup 开发组制定了以下标准：

## 一、提交内容

你需要提交以下内容：

- `.js` 后缀的拓展源文件
- `.svg` 后缀的拓展封面图(封面图必须为**600x300**)
- 拓展的元数据(包含拓展的名称、描述、作者、版本号等)
- `.json` 后缀的拓展翻译(也可以整合到拓展源文件中)
- `.md` 后缀的拓展说明文档(必要时可以选择，这里不做强制要求)

其中，拓展的元数据应放在拓展源文件的开头，详细信息参见[文档](https://docs.bilup.org/extensions/push-rules)

## 二、提交方法

### GitHub 仓库

你可以将拓展提交到 GitHub 仓库的 `extensions` 目录下。
(一般情况下你是没有提交权限的，所以你需要通过Pull Request来进行提交。)

### Bilup QQ 用户群
将拓展打包成 `.zip` 或 `.rar` 等压缩文件格式，然后发送到 QQ 群。(当然你也可以直接发送拓展文件)

发送之后记得 **@RyaninCn11** 或 **@DL_Grass**。

### 邮箱
我们并不推荐你通过这种方式来提交拓展。
你可以将拓展发送到 `support@bilup.org`。

## 三、采纳标准
以下是采纳标准：

- 必须保证按照规范制作(详细信息参见[文档](https://docs.bilup.org/extensions/push-rules))
- 必须不能包含**任何违规代码**
- 必须保证在当前版本的 Bilup 中能够正常运行
- 必须**有意义**
- 需要加入翻译

如果拓展符合以上所有标准(除了翻译部分)，那么它就会被采纳。
采纳后将会把您加入到鸣谢成员列表中(Bilup Extensions Galley)
